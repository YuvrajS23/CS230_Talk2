Introduction To GPU

A Graphics Processing Unit, popularly known as GPU, is a specialized processor designed to accelerate the rendering of images, videos, and animations. Originally developed for gaming, GPUs have become increasingly important in a variety of fields, including machine learning, scientific computing, and cryptocurrency mining. GPUs are highly parallelized and can perform thousands of operations simultaneously, making them much faster than traditional central processing units (CPUs) for tasks that require heavy computations. In short, GPUs are designed to provide better throughput & efficiency than CPUs. With the rise of artificial intelligence and big data, GPUs have become an essential tool for professionals and researchers who need to process vast amounts of data quickly and efficiently. And thus, the demand has touched a 3-year high in 2021 and is supposed to surpass CPUs in the following decade. Nowadays, Apple & other companies are incorporating more GPU cores constituting significant portions in their latest processors. But GPUs face several challenges in the real world that restrict us from using them indefinitely to optimize the processors' throughput.


Challenges to GPUs

As powerful as GPUs are, they face several challenges when it comes to their practical use in the real world. One of the most significant challenges is Scalability. GPUs require massive amounts of memory bandwidth, and as the number of processing cores on a GPU increases, the memory requirements grow exponentially. This growth has created a compute memory gap, where the amount of memory required by the GPU to perform computations far outstrips the amount of memory that can be provided within the same die size. This gap can be clearly observed to be increasing with time in both NVIDIA and AMD processors.

Alongside scalability issues, Security & Reliability also pose a significant challenge to GPUs. As GPUs become more widely used in critical applications like autonomous vehicles, healthcare, and finance, there is an increased risk of attacks and failures, which could have severe consequences. Therefore, it is highly important to develop & implement robust security and reliability measures to mitigate these risks. Before we talk about these security and reliability challenges in detail, we must first have a background on the GPU's underline architecture to understand it better.


Background on GPU Architecture

The architecture of a Graphics Processing Unit (GPU) is designed to accelerate the parallel execution of tasks that require massive amounts of computation, such as graphics rendering and scientific computing. The execution model of a GPU is based on a parallel processing approach that utilizes thousands of small processing cores to perform computations simultaneously. The processing cores are grouped into Streaming Multiprocessors (SMs), which are responsible for executing instructions in parallel. Each SM has its own set of resources, including shared memory, registers, and scheduling units. The warp scheduler assigns groups of 32 threads, called warps, to each SM, and the threads in a warp execute in lockstep to optimize performance. The GPU architecture also uses Memory Access Coalescing, which combines memory accesses from multiple threads into a single transaction, reducing memory latency and increasing memory throughput. Together, these features enable GPUs to achieve high levels of parallelism and performance, making them a powerful tool for a wide range of applications.

Security Challenges in GPU

Security challenges in GPUs are becoming increasingly important as they are being used in sensitive applications. Timing attacks are a form of side-channel attacks that exploit variations in execution time to reveal sensitive information like encryption keys. GPUs are particularly vulnerable to timing attacks due to their parallel architecture, which makes it challenging to synchronize the execution of threads. An attacker can use timing attacks to infer sensitive information like the AES key by measuring the execution time of the last round of the AES encryption algorithm on the GPU. By carefully selecting inputs, an attacker can measure the time taken by the GPU to access coalesced memory addresses and correlate it with the real encryption key used in the computation. Correlation turns out to be the highest for the real encryption key as shown by the red dot in the figure.

What is the solution to such threats over GPUs?

One solution that immediately pops up in someone's mind is the Naive solution that says that we must disable coalescing altogether and hence the correlation drops to approximately zero making our system robust to such attacks. But this solution degrades the performance of the GPU up to 178 percent which is certainly undesirable.

There are some other state-of-art solutions to the problems mentioned below:

  • Random Coalescing: introduces random delays in memory access on the GPU. This technique helps to reduce the precision of the timing measurements, making it more difficult for an attacker to infer sensitive information. However, Randomized Coalescing can introduce performance overhead and it may not be effective when the GPU is vulnerable to cache-based timing attacks or when caches and MSHRs(Miss Status Holding Registers) are shared between different processing units.
  • Software Solution: A software solution against timing attacks on GPUs is to use constant-time implementations of cryptographic algorithms. This approach ensures that the execution time of the algorithm is independent of the input and the secret key, making it difficult for an attacker to infer sensitive information. However, this approach addresses leakage only in AES applications and may not apply to other cryptographic algorithms.

Both of them have either performance on generality limitations. We, therefore, require performance-efficient general solutions to protect our system. Here comes a solution based on key insight over redundant data management (BCoal) to the rescue.

Bucketing-based Memory Coalescing (BCoal):-

BCoal is a bucketing-based memory coalescing technique that aims to improve the performance and security of GPUs by reducing the precision of timing measurements used in side-channel attacks. BCoal uses buckets to group memory accesses that occur within a certain time interval, thus reducing the variance in timing measurements. The number of buckets used in BCoal can be adjusted to balance security and performance, as a larger number of buckets can reduce the precision of timing measurements but can also increase the overhead of memory coalescing thus creating a security-performance tradeoff. BCoal also employs redundant data management to reduce the amount of data that needs to be processed, further improving the efficiency of the memory coalescing process. By improving the security and performance of GPUs, BCoal has the potential to enhance the usability of GPUs in a wide range of applications. BCoal is both security and performance efficient.


Reliability Challenges in GPU

GPU reliability challenges are becoming increasingly important as GPUs are being used in safety-critical and mission-critical applications. Sources of errors in GPUs can include manufacturing defects, thermal and power issues, and soft errors caused by cosmic radiation and electromagnetic interference. These errors can have several impacts including Silent Data Corruption (SDC) which leads to incorrect output. It can also result in severe accidents and hangs that can turn out to be very drastic in critical applications. The protection mechanisms employed to counter these errors include Error Checking and Correction (ECC) which can correct single-bit errors and detect double-bit errors and protects register file, caches, and memory. Other Techniques for protection include- Duplication/Triplication which means selective re-computation and Checkpointing. The problem with these current protection mechanisms is that protection against multi-bit errors is prohibitively expensive in GPUs and new techniques e.g. low-voltage GPU caches lead to high error rates. Thus, the current protection is not enough and is expensive. So what to do to make the protection mechanism more effective and efficient? A key insight: Redundant Data Management comes to the playground. It focuses on protecting only a fraction of memory. Now the question arises, Which fraction to protect? & How to protect it?

Let's answer the first question. The fraction of memory that should be protected must be Highly Accessed & Highly Shared. Let us term it Hot Memory. On Application Pattern Analysis as shown in the below figures, we find that a small fraction (Hot Memory) of memory is highly accessed and highly shared and the output is highly sensitive to faults in Hot Memory.

How to identify which fraction corresponds to Hot Memory? Source Code Profiling. Source code profiling is the process of analyzing a program's source code to identify performance bottlenecks and areas of inefficient memory usage, including hot memory.

Now let's answer the second question. Hot Memory can be protected by implementing a simple but effective replication mechanism that is Hot memory is replicated for detection/correction purposes. Also, GPU's latency tolerance feature helps in keeping the overheads low. This gives improved reliability at low-performance overhead.


Conclusion

GPUs have become an integral part of modern computing, and their demand is set to increase in the following decade due to their efficiency and high throughput. However, GPUs face several challenges, including scalability, security, and reliability. To optimize the performance of GPUs, we need to develop and implement robust security and reliability measures while maintaining performance. Random coalescing and constant-time implementations of cryptographic algorithms are some of the current solutions to timing attacks on GPUs. In the future, we will need to continue to innovate and develop new technologies to overcome these challenges and push the boundaries of computing further. Ultimately, GPUs are a powerful tool that can help us to tackle some of the world's most significant challenges, from climate change to disease research, and we need to ensure that we use them effectively and responsibly.
